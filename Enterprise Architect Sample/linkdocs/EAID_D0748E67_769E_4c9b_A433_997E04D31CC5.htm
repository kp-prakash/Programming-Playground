Scope

This section gives a definition of the scope for the testing process.  A broad scope is critical to the definition of the testing being proposed. This is to ensure that testing is performed on updates to code. It also ensures that there is not excessive testing of areas of the system that have not undergone change since the last rollout.

System Design

Brief overview of the systems design - outlining any different sections that may need to be tested and the context that they will be operable (i.e. Relationship between client and server as well as tests on the Client application, tests on the server side etc.)

Objectives

State here the intended results to be achieved from performing the tests. 

Roles

Identify the people that are responsible for:

  1. Writing the detailed test plan.

  2. Supervising the test procedures.

  3. Providing the resources for the test process.

  4. Documentation of the final results.

  5. Reworking of the test plan on completion.

Recurrence

Define what the frequency of recurring test will be (I.e. monthly, weekly, release only etc.).

Scenarios

This section is used to define the test scenarios for different features of the application that are requiring testing.   For each of these features there needs to be details defined as to the process of testing it.  The following is a list of categories that can be used to define these scenarios:

  • Define the feature of the application to be tested

  • Define the type of test to be used.

  • How it will be tested.
