Scenario Type Detail
Out of Stock on an item. Alternate
1. Use case begins when Storeroom worker selects the "Process Order" command.
2. This displays the list of items under order as well as the quantity available for each of these items.
3. Any Items that do not have sufficient quantity in-stock are highlighted.
4. User sets the entry as out of stock.
5. User checks if an order exists - if not add a stock order for the item.
6. User generates an Email stating the order is awaiting stock- this is and sent to the Client.
7. Processing message is displayed to the User.
Standard Order Process Basic Path
1. Use case begins when Storeroom worker selects the "Process Order" command.
2. This displays the list of items under order as well as the quantity available for each of these items.
3. Any Items that do not have sufficient quantity in-stock are highlighted.
5. User Collects the books 
6. User Prints the receipt
6. User passes books and receipt on to be packaged.
7 Email of order is generated and sent to the Client.
8. Order is stored locally for reference.
9. Processing message is displayed to the User.
